# UI Contracts: 団体メンバー・イベント管理ツール

**Phase**: 1 — Design
**Date**: 2026-05-27

---

## 画面一覧

| 画面 | パス | 主な機能 |
|------|------|----------|
| ダッシュボード | `/` | メンバー数・イベント数サマリー、直近イベント |
| メンバー一覧 | `/members` | メンバー一覧表示・フィルター・新規作成ボタン |
| メンバー新規作成 | `/members/new` | メンバー登録フォーム |

| イベント詳細 | `/events/[id]` | イベント情報・参加記録一覧 |
| イベント編集 | `/events/[id]/edit` | イベント情報編集フォーム |

## 画面コントラクト

### ダッシュボード (`/`)

**表示データ**:
- 在籍中メンバー数（ACTIVE）
- 今後30日以内のイベント数（SCHEDULED）
- 直近5件の予定イベント（タイトル・日時）

**アクション**:
- 「メンバー管理へ」ボタン → `/members`
- 「イベント管理へ」ボタン → `/events`

---

### メンバー一覧 (`/members`)

**表示データ**（テーブル形式）:
- 氏名 / 役割 / 連絡先 / 在籍状態 / 登録日

**フィルター**:
- 在籍状態：全員 / 在籍中 / 退会済み

**アクション**:
- 「新規メンバー追加」ボタン → `/members/new`
- 行クリック or 「編集」→ `/members/[id]/edit`
- 「削除」ボタン → 確認ダイアログ後に削除

**空状態**: メンバーが0人の場合は「メンバーが登録されていません」メッセージ + 新規作成ボタン

---

### メンバー新規作成・編集フォーム (`/members/new`, `/members/[id]/edit`)

**入力フィールド**:

| フィールド | 型 | 必須 | バリデーション |
|-----------|-----|------|--------------|
| 氏名 | テキスト | ✓ | 空文字不可 |
| メールアドレス | テキスト | △ | メール形式（email or phone 必須） |
| 電話番号 | テキスト | △ | （email or phone 必須） |
| 役割 | テキスト/セレクト | ✓ | デフォルト: 一般会員 |
| 在籍状態 | ラジオ/セレクト | ✓ | ACTIVE / INACTIVE |
| 備考 | テキストエリア | - | 最大500文字 |

**アクション**:
- 「保存」ボタン → バリデーション後保存、成功時 `/members` へリダイレクト
- 「キャンセル」ボタン → `/members` へ戻る

**エラー表示**: フィールド直下にインラインエラーメッセージ

---

### イベント一覧 (`/events`)

**表示データ**（テーブル形式）:
- イベント名 / 開催日時 / 場所 / ステータス / 参加登録者数

**フィルター**:
- ステータス：全て / 予定 / 終了 / 中止
- 期間：全て / 今後 / 過去

**アクション**:
- 「新規イベント作成」ボタン → `/events/new`
- 行クリック → `/events/[id]`（詳細）
- 「編集」→ `/events/[id]/edit`
- 「削除」ボタン → 確認ダイアログ後に削除

**空状態**: イベントが0件の場合は「イベントが登録されていません」メッセージ + 新規作成ボタン

---

### イベント新規作成・編集フォーム (`/events/new`, `/events/[id]/edit`)

**入力フィールド**:

| フィールド | 型 | 必須 | バリデーション |
|-----------|-----|------|--------------|
| イベント名 | テキスト | ✓ | 空文字不可 |
| 開催日時 | 日時ピッカー | ✓ | 有効な日時形式 |
| 終了日時 | 日時ピッカー | - | 開始より後 |
| 場所 | テキスト | - | 最大200文字 |
| 説明 | テキストエリア | - | 最大2000文字 |
| ステータス | セレクト | ✓ | SCHEDULED / COMPLETED / CANCELLED |

**アクション**:
- 「保存」ボタン → バリデーション後保存、成功時 `/events` へリダイレクト
- 「キャンセル」ボタン → `/events` へ戻る

---

### イベント詳細 (`/events/[id]`)

**表示データ**:
- イベント基本情報（名称・日時・場所・説明・ステータス）
- 参加記録テーブル: メンバー名 / 参加状況 / 備考

**参加記録操作**:
- メンバーを選択して参加状況（参加予定 / 欠席 / 未回答）を登録
- 既存記録の参加状況を変更
- 既存記録の削除

**アクション**:
- 「イベント編集」ボタン → `/events/[id]/edit`
- 「参加メンバー追加」→ メンバー選択モーダル + 参加状況選択

---

## 共通 UI コンポーネント契約

### ナビゲーション
- 左サイドバーまたはトップナビゲーション
- 「メンバー」「イベント」の2つのメインリンク + 「ダッシュボード」

### 確認ダイアログ
- 削除操作時に必ず表示
- 「削除する」（赤ボタン）/「キャンセル」の2択

### 空状態 (Empty State)
- アイコン + メッセージ + 主要アクションボタンの3点セット

### ローディング状態
- データ取得中はスピナーを表示

### エラー状態
- 操作失敗時はトースト通知でエラーメッセージを表示
